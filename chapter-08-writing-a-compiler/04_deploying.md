## Deploying the compiled contract

A compiler that only prints hex is a parlour trick. The payoff is putting that bytecode on a real chain and watching it run. This page walks the whole path from nothing to a live, callable contract on the Sepolia testnet, using only `openssl` and Foundry's `cast` from the command line. Every command here is one you can paste and run.

We use **Sepolia**, an Ethereum test network. The ETH on it is free and worthless, which is exactly what you want while learning. Never point these steps at mainnet with a key you generated in a tutorial.

### 1. Create a key

An Ethereum private key is nothing more exotic than a random 256-bit number on the secp256k1 curve, the same curve Bitcoin uses. `openssl` will generate one for us:

```sh
openssl ecparam -name secp256k1 -genkey -noout -out key.pem

# Pull the raw 32-byte private scalar out of the PEM file.
PRIV=$(openssl ec -in key.pem -text -noout 2>/dev/null \
  | grep -A3 'priv:' | tail -n +2 | tr -d ' :\n' | sed 's/^00//')
echo "0x$PRIV"
```

That hex string *is* the account. Whoever holds it controls the funds, so on mainnet you'd guard it like a password. The leading `00` strip handles a quirk of how OpenSSL serialises the number: it sometimes pads a leading zero byte, which we don't want in the 32-byte key.

### 2. Derive the address

The public address is derived from the key by elliptic-curve math and a keccak256 hash. You don't have to do that by hand; `cast` does it:

```sh
cast wallet address --private-key 0x$PRIV
# 0x13F439EF673E7A864A43e9dD401a25D53A793caC
```

This address is what you hand out to receive funds. It's safe to share; it reveals nothing about the private key.

### 3. Fund it

A fresh account has zero balance, and you can't deploy without gas to pay for it. On a testnet you get that gas from a **faucet**, a service that drips free test ETH to any address that asks. Search for a "Sepolia faucet," paste in your address, and request a small amount. A fraction of one test ETH is plenty; a deployment costs almost nothing.

Check the balance landed before going further:

```sh
RPC=https://ethereum-sepolia-rpc.publicnode.com
cast balance --rpc-url $RPC 0x13F439EF673E7A864A43e9dD401a25D53A793caC
```

A non-zero number means you're ready. That `RPC` variable is an **RPC endpoint**: a node someone else runs that will accept your transactions and answer your queries. You can run your own node, but for testing a public endpoint or a free Alchemy/Infura key is fine.

### 4. Deploy

Deployment is just a transaction with no recipient and the contract's **deploy bytecode** as its payload. That payload is exactly what our compiler prints, so we grab it straight from the compiler:

```sh
DEPLOY=$(python3 -c "from greeter_compiler import compile_source; \
  print('0x' + compile_source(open('Greeter.sol').read())[2].hex())")
```

With the bytecode in hand, the whole deploy is one command:

```sh
cast send --rpc-url $RPC --private-key 0x$PRIV --create $DEPLOY
```

`cast` signs the transaction with our key, sends it to the RPC, and waits for it to be mined. The receipt includes the new contract's address. Capture it:

```sh
ADDR=$(cast send --rpc-url $RPC --private-key 0x$PRIV --create $DEPLOY --json \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['contractAddress'])")
echo "deployed at $ADDR"
```

### 5. Call it

The contract is live. Reading from it is free (no transaction, just a query to the node), so calling our getter costs nothing:

```sh
cast call --rpc-url $RPC $ADDR 'greet()(uint256)'
# 42
```

There it is: the constructor seeded `greeting = 42`, and the bytecode *our* compiler produced returns it from real on-chain storage. Writing takes a transaction, because it changes state and must be mined:

```sh
cast send --rpc-url $RPC --private-key 0x$PRIV $ADDR 'setGreeting(uint256)' 99
cast call --rpc-url $RPC $ADDR 'greet()(uint256)'
# 99
```

### The whole thing, start to finish

Stripped of commentary, the entire journey from no account to a live contract is this:

```sh
openssl ecparam -name secp256k1 -genkey -noout -out key.pem
PRIV=$(openssl ec -in key.pem -text -noout 2>/dev/null \
  | grep -A3 'priv:' | tail -n +2 | tr -d ' :\n' | sed 's/^00//')

cast wallet address --private-key 0x$PRIV          # fund this address from a faucet

RPC=https://ethereum-sepolia-rpc.publicnode.com
ADDR=$(cast send --rpc-url $RPC --private-key 0x$PRIV --create $DEPLOY --json \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['contractAddress'])")

cast call --rpc-url $RPC $ADDR 'greet()(uint256)'  # 42
```

Six commands, and bytecode you generated by hand is running on a public blockchain.

That is the entire claim of this book made concrete: a contract we lexed, parsed, and code-generated ourselves, with no `solc` anywhere in the pipeline, is a real, callable thing on a real Ethereum network.
